# 需求文档

## 1. 应用概述

### 1.1 应用名称
AI视频创作平台

### 1.2 应用描述
一款基于AI技术的视频创作工具，支持用户通过文本输入或文件上传快速生成短视频内容。平台提供文案生成、语音合成、图片生成、视频合成等核心能力，帮助用户高效完成视频创作。

## 2. 用户与使用场景

### 2.1 目标用户
- 内容创作者：需要快速产出短视频内容的自媒体从业者
- 营销人员：需要制作宣传视频的市场推广人员
- 普通用户：希望制作个人视频的非专业用户

### 2.2 核心使用场景
- 用户输入主题或上传文案，快速生成配音配图的短视频
- 用户上传音频文件，根据音频内容生成匹配的视频画面
- 用户需要批量生成多个视频片段并合并为完整视频

### 3.2 页面功能说明

#### 3.2.1 注册/登录页面

**注册功能**
- 用户输入手机号、验证码、密码完成注册

**登录功能**
- 用户输入手机号、验证码完成登录

#### 3.2.2 首页

**创作模式选择**
- 提供三种创作模式：
  - 文案生成模式：用户输入主题，系统生成文案、语音、图片并合成视频
  - 音频生成模式：用户上传音频文件，系统根据音频生成图片并合成视频
  - 分段视频模式：用户输入多段文案或上传多个音频，系统生成多个视频片段供用户管理和合并
- 用户选择模式后进入对应的创作页面

**历史记录入口**
- 用户点击进入历史记录页面

#### 3.2.3 创作页面

**文案生成模式Tab**

- **文案输入区**

  - 用户选择语音类型（男声/女声）
  - 用户选择图片动态效果（缩放/平移/旋转/无）
  - 用户选择图片切换特效（淡入淡出/滑动/无）

- **生成按钮**
  - 用户点击生成按钮
  - 系统执行文案生成模式流程

**音频生成模式Tab（新增）**

- **音频上传区**
  - 用户上传音频文件（支持wav/m4a/mp3格式）
  - 显示已上传的音频文件名称

- **参数配置区**
  - 用户选择图片动态效果（缩放/平移/旋转/无）
  - 用户选择图片切换特效（淡入淡出/滑动/无）

- **生成按钮**
  - 用户点击生成按钮
  - 系统执行音频生成模式流程

**分段视频模式Tab（新增）**

- **分段文案输入区**

  - 用户选择图片动态效果（缩放/平移/旋转/无）

## 4. 业务规则与逻辑

### 4.1 用户注册登录规则

**注册规则**
- 手机号必须为11位数字
- 验证码必须为6位数字
- 密码长度为6-20位

**登录规则**
- 手机号验证码登录：验证码有效期为5分钟
- 同一手机号60秒内只能发送一次验证码

### 4.2 文案生成模式流程

1. 用户输入主题关键词
2. 系统调用AI文案生成能力，生成完整文案
3. 系统根据文案生成语音音频
4. 系统根据文案生成图片提示词
5. 系统调用图片生成能力，批量生成图片
6. 若所有图片生成成功，系统自动合成视频
7. 若有图片生成失败，系统停止自动合成，提示用户进入任务详情页面处理失败图片

### 4.3 音频生成模式流程（新增）

1. 用户上传音频文件（wav/m4a/mp3格式）
2. 系统调用short-speech-recognition技能识别音频内容，生成文案
3. 系统根据文案生成图片提示词
4. 系统调用图片生成能力，批量生成图片
5. 若所有图片生成成功，系统自动合成视频
6. 若有图片生成失败，系统停止自动合成，提示用户进入任务详情页面处理失败图片

### 4.4 分段视频模式流程（新增）

1. 用户输入多段文案或上传多个音频文件
2. 系统为每段文案/音频生成对应的视频片段：
   - 若为文案输入，系统生成语音音频
   - 若为音频上传，系统调用short-speech-recognition技能识别音频内容
   - 系统根据文案生成图片提示词
   - 系统调用图片生成能力生成图片
   - 系统合成该片段的视频
3. 用户在视频分段管理页面查看、编辑、排序片段
4. 用户点击合并视频按钮
5. 系统按顺序合并所有片段为完整视频

### 4.5 图片生成失败的容错与恢复机制

**失败检测与阻断**
- 批量生成图片过程中，若有一张或多张图片生成失败
- 系统停止后续的「合并视频」步骤，不执行自动合成操作

**用户提示**
- 系统向用户展示明确提示，告知哪些图片生成失败
- 提示用户进入「任务详情」页面，对失败图片进行单独重新生成
- 提示用户所有图片就绪后，可手动触发合并视频

**任务详情页容错展示**
- 保留失败图片的顺序位置（占位不消失）
- 保留该图片对应的文案/提示词文字
- 图片区域展示为空白或使用占位提示图（如「生成失败，点击重试」）

**单张重新生成**
- 用户在任务详情页面，可对失败的图片单独触发重新生成
- 用户可保持原提示词或修改提示词
- 系统调用图片生成重新生成该图片
- 生成完成后更新该图片

**手动合并视频**
- 所有图片生成成功后，任务详情页面显示「合并视频」按钮
- 用户点击按钮，系统根据图片、语音音频、图片动态效果、图片切换特效生成视频
- 生成完成后跳转至结果展示页面

### 4.6 语音合成规则

**语音类型**
- 男声：使用男性音色
- 女声：使用女性音色

**语音参数**
- 语速：正常语速
- 音量：标准音量

### 4.7 图片生成规则

**图片数量**
- 根据文案长度和音频时长自动计算图片数量
- 每张图片对应3-5秒的音频时长

**图片提示词生成**
- 系统根据文案内容生成图片提示词
- 每段文案对应一个图片提示词

**图片动态效果**
- 缩放：图片从小到大或从大到小缩放
- 平移：图片从左到右或从上到下平移
- 旋转：图片顺时针或逆时针旋转
- 无：图片静止不动

**图片切换特效**
- 淡入淡出：前一张图片淡出，后一张图片淡入
- 滑动：后一张图片从侧面滑入
- 无：直接切换

### 4.8 视频合成规则

**视频时长**
- 视频时长等于音频时长

**视频分辨率**
- 默认分辨率为1920x1080

**视频帧率**
- 默认帧率为30fps

**合成顺序**
- 按图片顺序依次合成
- 每张图片对应的音频片段同步播放
- 图片动态效果和切换特效按用户配置执行

### 4.9 分段视频合并规则（新增）

**合并条件**
- 所有分段视频必须生成成功
- 分段顺序由用户在视频分段管理页面确定

**合并逻辑**
- 系统按分段顺序依次拼接视频
- 保持每个分段的原始音频、图片、动态效果和切换特效
- 分段之间无额外转场效果

**合并结果**
- 生成一个完整视频文件
- 视频时长等于所有分段视频时长之和

### 4.10 历史记录规则

**记录保存**
- 所有任务记录永久保存
- 用户可随时查看历史记录

**任务状态**
- 排队中：任务已提交，等待处理
- 处理中：任务正在生成
- 已完成：任务生成成功
- 失败：任务生成失败

**排队位置**
- 排队中的任务显示当前排队位置和总排队数量
- 显示预估剩余时间

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户输入主题为空 | 提示用户输入主题 |
| 用户上传的音频文件格式不支持 | 提示用户上传支持的格式（wav/m4a/mp3） |
| 用户上传的音频文件为空 | 提示用户上传音频文件 |
| 音频识别失败 | 提示用户重新上传音频或手动输入文案 |
| 文案生成失败 | 提示用户重新生成或手动输入文案 |
| 语音合成失败 | 提示用户重新生成 |
| 图片生成失败 | 停止自动合成，提示用户进入任务详情页面重新生成失败图片 |
| 视频合成失败 | 提示用户重新生成 |
| 分段视频模式下用户未输入文案且未上传音频 | 提示用户至少输入一段文案或上传一个音频 |
| 用户删除所有分段片段 | 提示用户至少保留一个片段 |
| 分段视频合并时有片段生成失败 | 提示用户先完成所有片段的生成 |
| 用户未选择语音类型 | 默认使用女声 |
| 用户未选择图片动态效果 | 默认使用无动态效果 |
| 用户未选择图片切换特效 | 默认使用淡入淡出 |
| 网络异常导致任务中断 | 任务状态标记为失败，用户可重新生成 |
| 用户在排队中取消任务 | 任务从队列中移除，状态标记为已取消 |

## 6. 验收标准

1. 用户注册并登录成功
2. 用户选择音频生成模式，上传音频文件「example.mp3」
3. 系统识别音频内容，生成图片并自动合成视频（若图片全部生成成功）
4. 用户在结果展示页面预览视频
5. 用户下载视频文件

## 7. 质量与测试

### 7.2 CI 流水线

- 使用 GitHub Actions，每次 push/PR 到 main 分支自动运行
- 包管理器：pnpm（`pnpm install --frozen-lockfile` + `pnpm test`）
- 所有测试必须通过，失败则 CI 标记为失败

### 7.10.2 水印配置项

## 7.13 视频编辑功能（剪辑）

### 7.13.1 功能描述
用户在结果展示页面可对生成的视频进行裁剪操作，提取需要的片段。支持选择开始和结束时间，预览裁剪范围，生成裁剪后的视频并下载。

### 7.13.2 组件设计

**VideoClipEditor**
- Dialog 形式弹出，包含视频预览、时间轴选择器
- 支持双滑块选择裁剪范围（开始/结束时间）
- 实时预览裁剪区间
- 裁剪过程中显示进度条
- 裁剪完成后展示结果视频，支持预览和下载
- 支持取消操作，重置所有状态

**useVideoClip Hook**
- 管理裁剪状态（startTime/endTime/duration/isClipping/progress/error/clipResult）
- clipVideo(videoUrl)：执行裁剪，返回 Blob URL
- 使用 Canvas captureStream + MediaRecorder 实现浏览器端裁剪
- 支持中止操作（AbortController）
- 校验逻辑：开始时间 < 结束时间，结束时间 <= 视频时长

### 7.13.3 界面入口
- 单分视频结果页底部操作栏新增「剪辑」按钮
- 位于「下载视频」与「分享」之间
- 仅单视频模式显示，多片段模式不显示

### 7.13.4 输出格式
- 根据浏览器支持的编码自动选择（优先 MP4/H.264，回退 WebM/VP9）
- 视频质量 6Mbps 码率
- 保持原始分辨率

### 7.13.5 异常处理
- 开始时间 >= 结束时间：提示用户调整范围
- 结束时间超过视频时长：提示用户
- 网络加载失败：显示错误信息
- 裁剪取消：静默重置

### 7.15 B站视频发布功能

#### 7.15.1 功能描述
用户可在结果页一键将生成的视频发布到 B站（bilibili）。发布流程通过 orchestrator 的后台任务异步执行：先下载视频 → 上传到 B站 → 创建投稿。

#### 7.15.2 触发条件
- 视频已生成成功（有 video_url）
- orchestrator 已配置 B站 SESSDATA（通过 ProviderRouter 管理）
- 用户已配置 orchestrator 连接（API Key 或同域部署）

#### 7.15.3 界面入口
- 单分视频结果页底部操作栏，位于「分享」按钮之前
- 多片段模式：仅在有至少一个子视频完成时显示
- 按钮文案三态：「发布到B站」/「发布中...（禁用+旋转动画）」/「已发布（绿色勾）」

#### 7.15.4 交互流程
1. 用户点击「发布到B站」按钮
2. 前端调用 `publishVideo()` 函数 → POST `/api/jobs/publish-video`
3. orchestrator 创建 video_publish 任务（status: pending）
4. BackgroundTask 异步执行：下载视频 → B站 pre-upload → 分片上传 → create archive
5. 前端显示发布中状态（loading），防止重复提交
6. 发布成功/失败通过 toast 通知用户

#### 7.15.5 错误处理
- 网络错误：toast 提示错误信息
- orchestrator 未配置 B站 SESSDATA：返回 400 错误，前端展示失败原因
- 视频不存在/URL 过期：发布任务失败，jobs 表记录 error

#### 7.15.6 配置说明
- 环境变量 `VITE_ORCHESTRATOR_URL` 或 localStorage `orchestrator_url` 指定 orchestrator 地址
- localStorage `orchestrator_api_key` 保存 API Key（对应 orchestrator 的 `PO_API_KEY`）
- 开发环境默认使用 `/api`（生产同域）或 `http://localhost:8000`（开发跨域）

#### 7.15.7 依赖
- backend: orchestrator `POST /api/jobs/publish-video`（feature gate: video_publish）
- backend: orchestrator `services/bilibili_publisher.py` — B站上传完整流程
- backend: auth via JWT Bearer token 或 X-API-Key header
## 7.18 付费会员系统

### 7.18.1 功能描述
会员分级系统和功能权限控制。免费用户可使用基础功能，付费用户解锁高级创作能力。系统通过 orchestrator 后端管理订阅生命周期、用量配额和 feature gate 权限。

### 7.18.2 计划层级

| 计划 | 价格 | 配额 | 特色功能 |
|------|------|------|---------|
| 免费版 | ¥0 | 每日 3 次视频生成 | 基础分句、固定模板 |
| 基础版 | ¥9.99 | 每日 10 次视频生成 | 声音克隆、AI 优化 |
| 专业版 | ¥29.99 | 每日 50 次视频生成 | 批量分句、无水印 |
| 企业版 | ¥99.99 | 每日 200 次视频生成 | 全部高级功能、优先支持 |

### 7.18.3 Feature Gate 映射

| 功能 | 所需最低计划 | 说明 |
|------|------------|------|
| `voice_clone` | 基础版 | 声音克隆 / 高级语音 |
| `video_fixed_template` | 基础版 | 视频固定模板 |
| `batch_split` | 专业版 | 批量分句 |

### 7.18.4 前端组件

| 组件 | 用途 |
|------|------|
| `useOrchestratorMembership` Hook | 获取会员信息与用量配额，自动管理 JWT 生命周期 |
| `useFeatureGate` Hook | 功能级权限检测，配合 `RequireFeature` 组件做 UI 锁定 |
| `RequireFeature` 组件 | 包装高阶组件，不满足权限时显示 `LockedFeature` 占位 |

| `MembershipCard` | 个人主页会员卡片，展示计划名称与权益 |
| `MembershipUpgradeDialog` | 套餐对比 → 订单创建 → 支付确认 → 升级成功全流程 |
| `QuotaWidget` | 用量进度条（如 3/10），不足时显示预警 |
| `RouteGuard` | 路由级别的会员权限守卫 |
| `ApiSettingsDialog` 会员 Tab | 按会员等级限制 API Key 配置项可见性 |

### 7.18.5 后端集成

- **orchestrator `POST /api/payment/create-checkout`**: 创建订单
- **orchestrator `POST /api/payment/confirm-mock`**: 确认支付（开发阶段 mock）
- **orchestrator `POST /api/auth/subscription`**: 获取订阅信息
- **orchestrator `POST /api/auth/login`**: 自动注册/登录 shadow 账户
- **orchestrator Feature Gates**: `premium_content` 等 gate 控制后端功能暴露

### 7.18.6 健壮性设计

- Token 自动刷新：access_token 过期后自动用 refresh_token 续期
- Stale-while-revalidate：从 localStorage 展示缓存数据，后台异步刷新
- 401 自动重试：token 失效时清除并重新登录
- 降级：orchestrator 不可用时会员信息返回 null，功能按免费权限处理
- 配额不足预警：接近上限时 CreatePage 显示 amber 提示
