## Context

当前应用使用浅色主题，所有样式硬编码为浅色颜色值。随着用户量增长，夜间使用场景增多，需要支持深色模式以提升用户体验。

## Goals / Non-Goals

**Goals:**
- 实现系统级深色模式切换
- 保存用户主题偏好
- 所有组件支持深色模式
- 切换动画平滑

**Non-Goals:**
- 不实现自动跟随系统主题（后续迭代）
- 不实现定时切换功能
- 不改变现有布局结构

## Decisions

### 1. 使用 CSS 变量实现主题切换
**选择**: CSS 自定义属性 (CSS Variables)
**理由**: 
- 运行时切换无需重新编译
- 浏览器原生支持，性能好
- 易于维护和扩展

**替代方案**:
- SASS 变量：需要重新编译，不支持运行时切换
- JavaScript 内联样式：性能差，难以维护

### 2. 使用 React Context 管理主题状态
**选择**: React Context + useReducer
**理由**:
- 轻量级，无需额外依赖
- 与现有状态管理兼容
- 支持主题持久化

**替代方案**:
- Redux：引入不必要的复杂性
- Zustand：额外依赖，当前规模不需要

### 3. 使用 localStorage 持久化
**选择**: localStorage
**理由**:
- 简单可靠，无需后端支持
- 数据持久化在客户端
- 性能好，同步操作

**替代方案**:
- Cookie：数据量限制，安全性问题
- SessionStorage：标签页关闭后丢失

## Risks / Trade-offs

- **[风险]** 第三方组件库可能不支持深色模式 → **缓解**: 使用 CSS 变量覆盖组件样式
- **[风险]** 用户可能忘记已启用深色模式 → **缓解**: 提供明显的切换按钮
- **[权衡]** 增加包大小约 2KB → 可接受，用户体验提升明显

## Migration Plan

1. 创建主题上下文和提供者
2. 重构样式为 CSS 变量
3. 添加切换按钮组件
4. 逐个组件适配深色模式
5. 测试和调试
