# 图片轮播视频功能模块设计方案

## 一、总体功能架构

### 1.1 模块划分

```
┌─────────────────────────────────────────────────────────────┐
│                    图片轮播视频功能模块                         │
├─────────────┬─────────────┬─────────────┬───────────────────┤
│  轮播播放器  │  视频播放器  │ AI生图管理   │    任务总控中心    │
├─────────────┼─────────────┼─────────────┼───────────────────┤
│ • 过渡特效  │ • 播放/暂停 │ • 模型列表   │ • 创建 → 生成图片  │
│ • 切换时间  │ • 进度控制  │ • 切换模型   │ • 预览 → 生成视频  │
│ • 循环模式  │ • 音量控制  │ • 参数配置   │ • 导出/下载       │
│ • 动态效果  │ • 全屏支持  │ • 历史记录   │ • 重新生成图片   │
└─────────────┴─────────────┴─────────────┴───────────────────┘
```

### 1.2 数据流架构

```
用户输入文案 ──→ TTS语音合成 ──┐
                              │
用户选择图片数量 ──→ AI生图模型 ──→ 生成图片列表 ──→ 轮播预览 ──→ 合成视频
                              │                        ↑
                              └─→ 存入数据库(gallery_images) ───┘
```

### 1.3 核心数据模型

| 表名 | 用途 |
|------|------|
| `video_tasks` | 主任务记录，包含模式、状态、参数、tts_audio_url、tts_duration_seconds 等 |
| `gallery_images` | 图片记录，关联任务、顺序、英文提示词(prompt)、中文原文(original_prompt) |
| `user_settings` | 用户模型配置（内置/自定义） |

## 二、轮播播放器模块设计

### 2.1 功能规格

| 功能项 | 规格 | 默认值 |
|--------|------|--------|
| 切换时间 | 每张图片停留时长 | 根据语音时长自动分配 |
| 过渡效果 | 图片间切换动画 | 渐隐渐显 |
| 循环模式 | 是否循环播放 | 单次播放（视频导出用） |
| 动态效果 | 图片本身的运动效果 | 慢慢放大 |

### 2.2 过渡效果清单

| 效果名 | 描述 | 实现方式 |
|--------|------|----------|
| 直接切换 | 瞬间切换，无动画 | 直接替换图片 |
| 渐隐渐显 | 前图淡出，后图淡入 | CSS opacity 过渡 |
| 左滑 | 前图向左滑出，后图从左滑入 | CSS translateX |
| 右滑 | 前图向右滑出，后图从右滑入 | CSS translateX |
| 上滑 | 前图向上滑出，后图从上滑入 | CSS translateY |
| 下滑 | 前图向下滑出，后图从下滑入 | CSS translateY |
| 翻页 | 3D 翻页效果 | CSS rotateY + perspective |
| 缩放淡出 | 前图缩小淡出，后图放大淡入 | CSS scale + opacity |

### 2.3 动态效果清单（Ken Burns）

| 效果名 | 描述 | 实现方式 |
|--------|------|----------|
| 无效果 | 静态展示 | 无变换 |
| 慢慢放大 | 图片逐渐放大 | CSS scale 从 1 到 1.15 |
| 慢慢缩小 | 图片逐渐缩小 | CSS scale 从 1.15 到 1 |
| 向左平移 | 图片向左缓慢移动 | CSS translateX 位移 |
| 向右平移 | 图片向右缓慢移动 | CSS translateX 位移 |
| 向上平移 | 图片向上缓慢移动 | CSS translateY 位移 |
| 向下平移 | 图片向下缓慢移动 | CSS translateY 位移 |

### 2.4 播放模式

| 模式 | 行为 |
|------|------|
| 预览模式 | 用户点击"预览"，在页面内实时播放轮播，带过渡效果 |
| 合成模式 | 点击"生成视频"，使用 Canvas + MediaRecorder 录制为 WebM |
| 循环播放 | 预览模式下可开启/关闭循环 |

## 三、视频播放器模块设计

### 3.1 功能规格

| 功能 | 描述 |
|------|------|
| 播放/暂停 | 点击按钮或视频区域切换 |
| 进度条 | 显示当前播放位置，支持拖拽跳转 |
| 音量控制 | 0~100% 音量滑块，支持静音 |
| 全屏播放 | 切换全屏模式 |
| 播放速度 | 0.5x / 1x / 1.5x / 2x |
| 时长显示 | 当前时间 / 总时长 |

### 3.2 组件结构

```

  ├─ <video> (原生视频元素)

  │   ├─ <ProgressBar>
  │   ├─ <PlayPauseButton>
```

## 四、AI生图模型管理模块设计

### 4.1 设计目标

- 统一入口：所有图片生成功能都从"AI生图"功能入口进入
- 模型透明：用户不需要关心底层调用的是 Jimeng 还是 Vidu
- 配置灵活：支持内置模型和自定义模型切换
- 一键使用：点击即可开始生成，无需复杂配置

### 4.2 模型来源架构

```
AI生图功能入口
    │
    ├─→ 内置AI模型（默认）
    │       └─→ 系统调用 可灵(Kling) 内置技能
    │
    ├─→ 自定义模型（用户在设置中配置）
    │       ├─→ 用户 Vidu API Key
    │       └─→ 用户 Jimeng API Key（Jimeng 不支持图片生成，自动回退到可灵）
    │
    └─→ 提示词自动优化
            └─→ 系统调用 文心大模型(ERNIE 4.5 Turbo) 将中文文案优化为英文生图提示词
```

### 4.3 模型管理界面

在创作页面（CreatePage）的"图片轮播视频"模式下，AI生图功能以以下方式整合：

**方式一：实时生图（默认）**
- 用户填写语音文案 → 点击"立即创作"
- 系统自动：①生成语音 ②分断文案 ③调用AI生图模型生成多张图片
- 用户直接进入图片管理页

**方式二：独立生图入口**
- 在图片管理页（GalleryPage）顶部添加"AI生图"按钮
- 点击后弹出模型选择器
- 用户可选择：重新生成全部 / 重新生成单张 / 生成新图片

### 4.4 模型选择器设计

```
┌──────────────────────────────────────┐
│         AI生图模型选择器               │
├──────────────────────────────────────┤
│ 当前模型：系统内置（Jimeng）            │
│                                      │
│ [切换模型]                           │
│ ┌──────────────────────────────────┐ │
│ │ 系统内置                         │ │
│ │   ○ Jimeng 生图（默认）          │ │
│ │   ○ Vidu 生图                    │ │
│ │                                  │ │
│ │ 自定义模型（需先在设置中配置）     │ │
│ │   ○ 我的 Jimeng API              │ │
│ │   ○ 我的 Vidu API                │ │
│ └──────────────────────────────────┘ │
│                                      │
│ [开始生成]                           │
└──────────────────────────────────────┘
```

### 4.5 生图参数配置

| 参数 | 选项 | 说明 |
|------|------|------|
| 图片数量 | 自动计算 | 根据语音合成时长自动计算：每6秒1张，最少1张，上限8张 |
| 分辨率 | 720x1280 / 1080x1920 等 | 与视频分辨率一致 |
| 参考图片 | 可选上传 | 用于图生图模式 |
| 提示词优化 | 自动（LLM） | 系统调用文心大模型将中文文案自动优化为高质量英文生图提示词 |

### 6.2 关键交互说明

**步骤1：模型检测**
- 进入"图片轮播视频"模式时，系统检测用户是否配置了可用的生图模型
- 未配置时弹窗引导用户到设置页配置（支持内置AI或自定义API）

**步骤2：语音生成**
- 用户输入文案后，必须先进行语音合成（或试听）
- 语音时长决定了轮播视频的总时长

**步骤3：图片生成**
- 系统根据文案分断数量确定图片数量
- 每段文案调用一次生图API
- 所有图片并行生成，轮询等待完成
- 生成完成后自动存入 gallery_images 表

**步骤4：图片管理**
- 用户可以预览当前所有图片
- 每张图片显示中文原文描述（original_prompt），而非英文提示词
- 点击"修改提示词并重新生成"可编辑：
  - 中文描述（显示用，保存到 original_prompt）
  - 英文提示词（发给AI生图模型，保存到 prompt）
- 支持预览轮播效果（实时渲染，不生成视频文件）

**步骤5：视频合成**
- 用户确认图片满意后，点击"生成轮播视频"
- 使用 Canvas + MediaRecorder 逐帧渲染
- 同时混入已生成的语音音频
- 输出 WebM 格式视频文件，上传至 `generated-videos` Storage bucket

**步骤6：结果展示**
- 在结果页展示生成的视频
- 支持播放控制、下载、分享

**步骤7：历史记录入口**
- 所有图片轮播视频任务在历史记录中可点击直接进入图片管理页面
- 视频合成失败时，历史记录显示"可手动合成"状态，用户可点击进入图片管理重新合成
- 已合成语音的任务支持在历史记录中直接下载语音文件

## 七、技术架构

### 7.1 前端技术栈

| 技术 | 用途 |
|------|------|
| React + TypeScript | UI 框架 |
| HTML5 Canvas | 轮播渲染 + 视频录制 |
| MediaRecorder API | 视频录制为 WebM |
| Web Audio API | 音频混音（语音 + BGM） |
| Tailwind CSS | 样式系统 |
| shadcn/ui | 组件库 |

### 7.2 后端技术栈

| 技术 | 用途 |
|------|------|
| Supabase Edge Functions | API 代理调用（Vidu、可灵、文心LLM、TTS等） |
| Supabase Storage | 图片/视频存储（`generated-media`、`generated-videos`、`generated-audio`） |
| Supabase Database | 任务和图片数据持久化 |
| 可灵(Kling) 内置技能 | 图片生成（默认内置AI） |
| Vidu API | 图片/视频生成（自定义API模式） |
| 文心大模型(ERNIE 4.5 Turbo) | 提示词自动优化 |
| MiniMax TTS | 语音合成 |

### 7.3 视频合成技术方案

```
Canvas 逐帧渲染
    │
    ├─→ 计算每帧进度 (基于目标帧率 30fps)
    ├─→ 根据时间线确定当前显示哪张图片
    ├─→ 应用 Ken Burns 动态效果 (scale/translate)
    ├─→ 应用过渡效果 (两张图片间交叉淡入淡出或滑动)
    ├─→ ctx.drawImage() 绘制到 Canvas
    ├─→ canvas.captureStream() 获取视频流
    ├─→ MediaRecorder 录制为 WebM Blob
    └─→ 上传 Supabase Storage
```

### 7.4 音频混音方案

```
语音 TTS 音频
      │
      ├─→ AudioContext.decodeAudioData() → AudioBuffer
      │
BGM 音频                         ┌─→ 语音轨道（100% 音量）
      │                          │
      ├─→ decodeAudioData() ────┼─→ BGM 轨道（用户设置音量）
      │                          │
      └─→ 时长对齐（循环/截断）    └─→ OfflineAudioContext 混音
                                          │
                                          ▼
                                    输出混音 AudioBuffer
                                          │
                                          ▼
                                    转为 WAV Blob
                                          │
                                          ▼
                                    与视频流合并（或分别处理）
```

### 7.5 关键性能优化

| 优化点 | 方案 |
|--------|------|
| 图片预加载 | 所有图片在渲染前完成加载，避免播放时卡顿 |
| Canvas 分辨率 | 限制最大宽度 1920px，等比缩放高度 |
| 帧率控制 | 预览模式 30fps，录制模式 30fps |
| 并行生成 | 多张图片同时调用 API，减少总等待时间 |
| 分段处理 | 长文案自动分断为 30 字以内的片段，每段对应一张图 |

## 八、边界情况处理

| 场景 | 处理方案 |
|------|----------|
| 生图失败 | 标记该图片为失败，允许用户单独重试；部分失败时继续使用成功图片合成视频 |
| 网络中断 | 本地保存进度，恢复后自动轮询 |
| 视频合成失败 | 图片已保存至 gallery_images，用户可进入图片管理手动重新合成 |
| 提示词优化失败 | 降级为使用原始中文文案作为生图提示词 |
| 匿名用户操作 | gallery_images RLS 策略允许 user_id 为 null，支持未登录用户使用 |
| 浏览器不支持 MediaRecorder | 降级为仅提供轮播预览，提示用户浏览器限制 |
| 用户中途离开页面 | 任务在后台继续执行，用户可从历史记录点击进入图片管理继续操作 |

### 9.1 未来可扩展功能

|------|------|

### 9.2 模型配置扩展

当前模型配置结构：

```typescript
interface ModelConfig {
  llm:  { source: 'builtin'|'custom', provider: string, apiKey?: string };
  tts:  { source: 'builtin'|'custom', provider: string, apiKey?: string };
  video:{ source: 'builtin'|'custom', provider: string, apiKey?: string };
  image:{ source: 'builtin'|'custom', provider: string, apiKey?: string };
}
```

新增模型只需：
1. 在设置界面添加新的 provider 选项
2. 创建对应的 Edge Function
3. 在视频/图片服务层增加 provider 判断逻辑
