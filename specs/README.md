# 规范驱动开发（Spec-Driven Development）指南

本项目采用规范驱动开发流程，所有新功能在编码前必须先完成需求文档和设计文档。

## 文件结构

```
specs/
├── README.md           # 本指南
├── requirements.md     # 项目整体需求文档
├── design.md           # 架构设计文档
└── feature-xxx.md      # 单个功能的需求/设计文档（开发中创建）
```

## 工作流程

### 新增功能时的三步流程

```
1️⃣ 写需求 → 2️⃣ 做设计 → 3️⃣ 写代码
```

#### 第 1 步：生成需求文档

在编辑器 AI 助手中输入：

> 我要开发一个新功能：[功能描述]。请使用 spec-driven 的 `generate-requirements` 工具生成需求文档。

AI 会调用 MCP 工具，在 `specs/` 下生成需求文档。

**示例**：
> 我要给视频创作工具添加"批量导入图片"功能，用户可以一次性上传多张图片并自动分段生成轮播视频。

#### 第 2 步：生成设计文档

需求确认后，在 AI 助手中输入：

> 基于刚才的需求文档，请使用 `generate-design-from-requirements` 生成设计文档。

AI 会读取 `specs/requirements.md`，生成 `specs/design.md`。

#### 第 3 步：生成代码

设计确认后，在 AI 助手中输入：

> 基于设计文档，请使用 `generate-code-from-design` 生成实现代码。

AI 会读取 `specs/design.md`，输出可运行的代码。

## 与 AI 助手协作的快捷方式

如果你在使用我（秒哒 AI）进行开发，可以直接告诉我：

> "我要开发 [功能名]，请按规范驱动流程走：先写需求，再出设计，最后写代码。"

我会自动按以下步骤执行：
1. 和你确认需求细节
2. 更新 `specs/requirements.md`
3. 编写/更新 `specs/design.md`
4. 按设计文档实现代码

## 注意事项

- **需求文档是底线**：需求文档中的每一条都必须被设计文档覆盖，设计文档中的每一条都必须被代码实现
- **可追溯性**：如果发现 bug，优先检查是需求遗漏、设计缺陷还是实现错误
- **文档即契约**：需求文档是产品和开发之间的契约，确认后不轻易变更
