# 视频创作工具 — 需求文档

> 格式：EARS（Easy Approach to Requirements Syntax）

## 1. 概述

### 1.1 产品定位
一款基于 AI 的在线视频创作工具，支持文字/图片生成视频，结合语音合成和背景音乐，让用户轻松创作带音频的精彩内容。

### 1.2 目标用户
- 内容创作者（短视频、口播、广告）
- 自媒体运营者
- 需要快速生成视频素材的企业用户

---

## 2. 功能需求

### 2.1 视频生成

#### FR-001 文生视频
When 用户在创作页选择"文生视频"模式并输入视频描述文本，  
The 系统 SHALL 调用 AI 视频生成模型，根据文本描述生成对应视频，  
Where 用户可选择视频尺寸（720x1280、1280x720 等）和时长（5/8/10 秒）。

#### FR-002 图生视频
When 用户在创作页选择"图生视频"模式并上传参考图片，  
The 系统 SHALL 调用 AI 视频生成模型，基于参考图片生成动态视频。

#### FR-003 视频 Remix
When 用户上传已有视频并输入编辑说明，  
The 系统 SHALL 调用 AI 模型对视频进行局部编辑或风格转换，  
Where 编辑说明必须为非空文本。

#### FR-004 图片轮播视频
When 用户在创作页选择"图片轮播视频"模式并输入口播文案，  
The 系统 SHALL：
1. 调用 AI 生成多张与文案匹配的图片
2. 调用语音合成将文案转为口播音频
3. 将图片与音频合成为带转场效果的轮播视频

#### FR-005 批量图片生成
When 系统根据口播文案分句后，  
The 系统 SHALL 为每个分句并行生成一张对应的图片，  
Where 每张图片独立生成，支持单独重新生成。

### 2.2 语音合成

#### FR-006 TTS 语音合成
When 用户输入文案并选择音色/语速/音量/音调参数，  
The 系统 SHALL 调用 MiniMax TTS API 将文本转为 MP3 音频，  
Where：
- 支持内置中文男声/女声音色
- 支持豆包克隆音色
- 支持用户自定义克隆音色

#### FR-007 语音试听
When 用户点击"试听"按钮，  
The 系统 SHALL 生成音频并自动播放，  
Where 第二次点击时复用已生成音频，不重复调用 API。

#### FR-008 音色克隆
When 已登录用户上传/录制 10-300 秒的音频样本并命名音色，  
The 系统 SHALL 调用克隆 API 创建自定义音色，  
Where 克隆成功后音色可用于后续语音合成。

### 2.3 背景音乐

#### FR-009 背景音乐配置
When 用户在创作页开启背景音乐选项，  
The 系统 SHALL 将用户上传的背景音乐与口播音频混合，  
Where：
- 背景音乐音量可调（0-100%）
- 混合后的音频自动上传至存储

### 2.4 字幕配置

#### FR-010 字幕叠加
When 用户在图片轮播视频模式下开启字幕，  
The 系统 SHALL 将口播文案按句子分段，叠加到视频对应片段上，  
Where 支持字体、字号、样式选择。

### 2.5 图片效果

#### FR-011 图片动效
When 用户选择图片效果（放大/缩小/平移/旋转等），  
The 系统 SHALL 在轮播视频的每张图片上应用对应的 CSS/Canvas 动画，  
Where 效果持续时间为该图片在视频中的展示时长。

#### FR-012 转场效果
When 用户选择转场方式（渐隐/滑动等），  
The 系统 SHALL 在图片切换时应用对应的过渡动画。

### 2.6 任务管理

#### FR-013 任务创建与跟踪
When 用户点击"开始生成"，  
The 系统 SHALL 创建视频任务并跳转至进度页，  
Where 任务状态包括：排队中、生成中、合成中、完成、失败。

#### FR-014 历史记录
When 用户访问"历史"页面，  
The 系统 SHALL 展示所有历史任务列表，  
Where 支持按状态筛选和查看详情。

### 2.7 用户系统

#### FR-015 用户登录与注册
When 未登录用户访问需要消耗算力的功能，  
The 系统 SHALL 拦截并提示用户先登录，  
Where 支持手机验证码、用户名密码、微信三种登录方式。

#### FR-016 登录拦截
When 未登录用户点击"开始生成"/"试听"/"重新生成"/"生成视频"等按钮，  
The 系统 SHALL 弹出提示并跳转至登录页，  
Where 登录成功后自动返回原页面。

#### FR-017 API 密钥管理
When 已登录用户进入设置页，  
The 系统 SHALL 允许用户配置自己的第三方 API 密钥（豆包、即梦、MiniMax 等），  
Where 密钥仅存储在用户自己的数据库记录中。

## 3. 非功能需求

### 3.1 性能
- NFR-001：页面首屏加载时间 < 3 秒
- NFR-002：语音合成 API 响应时间 < 10 秒
- NFR-003：视频生成任务排队状态每 5 秒自动刷新

### 3.2 安全
- NFR-004：所有 API 调用必须经过登录校验（消耗算力的功能）
- NFR-005：用户上传的文件必须经过类型和大小校验（图片 ≤10MB，视频 ≤50MB）
- NFR-006：用户 API 密钥加密存储，不在客户端暴露

### 3.3 可用性
- NFR-007：系统 SHALL 在所有错误场景下显示中文错误提示
- NFR-008：登录/注册错误 SHALL 将 Supabase 英文错误翻译为中文

### 3.4 兼容性
- NFR-009：支持桌面端和移动端浏览器访问
- NFR-010：支持主流浏览器（Chrome、Safari、Edge）

### 2.8 图片生成与重新生成

#### FR-018 单张图片重新生成
When 用户在任务详情页点击某张图片的「修改提示词并重新生成」按钮，修改提示词后点击「确认重新生成」，
The 系统 SHALL：
1. 先将修改后的提示词保存到数据库，即使后续生成被取消也保留修改
2. 调用图片生成模型重新生成该图片
3. 在生成过程中显示进度条（起始 5%，提交完成后 15%，轮询期间动态增长）并提供「取消生成」按钮
4. 生成成功后更新图片并刷新列表，失败则标记该图片为失败状态并展示错误信息

#### FR-019 生成进度保持
When 用户在图片重新生成过程中离开任务详情页（如返回历史列表），再次进入同一任务详情页时，
The 系统 SHALL 自动检测并恢复未完成的图片生成任务，继续轮询进度并同步更新 UI，
Where 恢复机制基于 sessionStorage 记录，仅恢复 10 分钟内的任务。

#### FR-020 图片生成取消
When 用户在图片重新生成过程中点击「取消生成」按钮，
The 系统 SHALL 立即停止轮询并释放相关资源，
Where 已保存的提示词修改不受影响。

### 2.9 提示词优化与预览

#### FR-021 图片生成提示词预览
When 用户在 API 设置的「图片模型」选项卡中点击「预览效果」按钮，
The 系统 SHALL 弹出预览对话框，提供以下能力：
1. 输入原文（最多 500 字）
2. 点击「自动优化提示词」后，调用 LLM 将原文优化为高质量英文生图提示词
3. 优化后的提示词展示在可编辑文本框中，用户可手动调整
4. 点击「生成图片」后调用当前选中的图片模型生成预览图片

#### FR-022 提示词优化策略自定义
When 用户在预览对话框中点击「修改提示词优化策略」，
The 系统 SHALL 展开策略编辑器，允许用户：
1. 在大文本框中编辑自定义的 System Prompt 策略
2. 上传 .md 或 .txt 文件导入策略内容
3. 点击「恢复默认」还原为内置策略
4. 点击「查看示例」跳转到标准策略示例页面
Where 自定义策略通过 localStorage 持久化，在后续优化操作中生效。

#### FR-023 提示词优化服务端策略
When 用户调用提示词优化接口时，
The 系统 SHALL 优先使用用户自定义的策略（通过 optimize-prompt Edge Function 的 systemPrompt 参数传入），
Where 未自定义时使用内置默认策略。

#### FR-024 图片生成提示词策略升级至V10
When 系统执行任何图片生成提示词优化时，
The 系统 SHALL 使用内置的 V10（VisionCraft）策略，
Where 该策略包含：
- 对话式引导模式（5轮提问，从模糊想法到清晰视觉描述）
- 视觉知识图谱（自动扩展文化概念为视觉元素）
- 意图解析引擎（提取主体/场景/动作/风格/构图/光影6维度）
- 古代/现代分流管线（时代检测→朝代检测→差异化提示词生成）
- 多样化随机选择（10视角/8构图/10风格随机组合，替代固定轮转）
- 结构化视觉元素（建筑/服装/道具/氛围按上下文智能筛选）
- 风格管理模块（支持风格包应用与混合）
- 批量生成（变量维度矩阵：场景×时间×风格=N张图）

## 4. 技术约束

- TC-001：前端使用 React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui
- TC-002：后端使用 Supabase（PostgreSQL + Edge Functions + Storage）
- TC-003：视频生成依赖第三方 AI 模型 API（Sora / Vidu / 即梦 / 豆包）
- TC-004：语音合成依赖 MiniMax TTS API
- TC-005：图片生成依赖可灵 / 豆包 / Vidu / 商汤 SenseNova 图像生成 API
- TC-006：提示词优化依赖内置 LLM Gateway（通过 optimize-prompt Edge Function）
- TC-007：跨页面状态保持使用 sessionStorage，策略配置使用 localStorage
- TC-008：图片生成提示词优化策略版本为 V10（VisionCraft），核心模块位于 `src/lib/history-prompt.ts`，所有涉及图片生成的文字均通过该策略优化
